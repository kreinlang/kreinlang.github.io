Something about krein as a language.

Here are some questions normally asked about the language from friends and one for our communication platforms.

Speed

Krein is not a fast language but this is being worked on, this project is still in beta version so as time passes expect things like speed to have increased greatly since it is even on of the aims for creating the language

Extensive

Krein can utilize power that was not even offered in the main build, this is through modulization or using code that is out of your project written by someone else or yourself. This also helps in large project handling.

Error & bugs

Try catch methods are included in the package alhtough there function is not yet revised for better functioning. But if you write good code that can handle its errors in form of validation, you are very safe to use the language.
